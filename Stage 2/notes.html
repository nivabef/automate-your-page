Intro to Programming

Lesson One: The basics of the web and HTML

Thinking like a programmer

  1. Procedural thinking: step 1, step 2, step 3
  2. Abstract thinking: 1+1=2, whether is is apples or oranges or houses you're counting.
  3. Technical empathy: Thinking like a computer.
  4. Systems thinking: Breaking down big ideas into pieces.

HTML: hypertext markup language

HTML elements are pieces of HTML documents. They are indicated with HTML tags.

Unlike humans, computers can't mentally fill in gaps if there are errors or information missing. A human has to be very careful there are no mistakes when giving directions to a computer.

HTML elements are either inline or block. Block elements form an invisible box around the content inside them.

Lesson Two: Creating a structured document with HTML

House analogy

HTML files - the layout of the house, like where the walls are

CSS files - the color of the carpeting, how the house is decorated

Javascript - interactive elements of the house, such at the TV remote and garage door opener

HTML language

HTML is a language with syntax and rules. It uses tags, which are like elements of a tree.

DOM: document object model

DOM is a standard convention for representing and interacting with elements in HTML.

Each HTML tag creates an element in the DOM that the browser uses to display the page.

CSS

CSS allows you to use syntax and rules to change how elements look on the page.

Lesson Three: Adding CSS style to HTML structure

Avoiding repetition

CSS lets programmers avoid repetition by assigning certain styles to all instances of a specific tag in the HTML.

This also allows programmers to avoid errors. For example, using CSS to make all text in h1 tags red, you won't accidentally make one h1 tag blue and make your web page inconsistent.

CSS: cascading style sheets

Cascading means that the most specific rule is applied to every element.

In real life, national laws > state laws > municipality laws > your household rules. If they were cascading, your household rules would also apply to national law.

Lesson Five: Variables and strings

Variables

Variables are names given to values. For example, you could assign "35" to the variable "my_age". The next time the variable "my_age" is mentioned, the computer knows that it equals 35.

Strings

Strings are specific lines of text. They are assigned to variables, just like numbers are. For example, a variable could contain the text "This is an example of a string." Strings can be assigned by putting the text into either single or double quotation marks.

Lesson Six: Functions (aka procedures)

What are they?

Functions/procedures take input and then do something with the input to produce output. For example, if you want the function to add two numbers together, you would input the two numbers and then the function would work its magic to output the sum of the numbers.

Using paramaters to define the output of a function

To make a function that simply adds two numbers together and outputs the sum, you could do this:

def sum(x,y):
answer = x + y
return answer

This returns (outputs) x + y. You could also just code it like this:

def sum(x,y):
return x + y

That would also return (output) x + y.

You must use a return statement. Otherwise, the function won't define an output.

Lesson Seven: If and while

Boolean values

There are two boolean values: True and False. A computer program often has to decide the boolean value of a statement.

Types of procedures

If procedures. This one returns the absolute value of a number by changing the sign of the number only if it's negative.

if x<0:
x=-x
return x


Else procedures. With these, if the first statement is False, the computer will perform the second instruction.

if (test expression):
(Do this thing)
else:
(Do this other thing)


Or procedures. The value comes back True if either one of the two things is True.

if (expression 1) or (expression 2):
(Do this thing)


Loops. The cycle will keep looking until the value doesn't meet the statement.

i=0
while i<5:
print i
i=i+1


Breaks. This is a way to stop the loop even when the test condition is true.

i=1
while True:
if i>n:
break
print i
i=i+1

Lesson Eight: Structured data

Lists

A list is a sequence of anything, unlike strings which can only be a sequence of characters. The sequence could be characters, numbers, strings or other lists.

list examples:
p=['y','a','b','b','a']
colors=['red','yellow','green','blue']
numbers=[1,2,3,4,5,6]

Lists can be nested within other lists and can be of mixed type:
mixed_list=['yellow', 4, ['f', 'table']]

Changing lists

Mutation. This would change the item indexed at list position 0 and leave the rest of the list items as they are:
p=['h','e','l','l','o']
p[0]='y'

Aliasing is referring to the same thing by two different names. For example, if you made p=q, then p and q would become the same. If you later changed the value of one, it would change the value of both.

Appending items to lists:
stooges=['Moe','Larry','Curly']
stooges.append('Shemp')
This would add 'Shemp' to the end of the stooges list, so it now contains four items.
